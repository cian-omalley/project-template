# Release Checklist

## Before Release

- [ ] Project brief is updated.
- [ ] Tasks are reviewed.
- [ ] Changelog is updated.
- [ ] Setup guide is accurate.
- [ ] Documentation is complete enough for a new user.
- [ ] Secrets/private files are not committed.
- [ ] Build/export has been tested.

## Release Package

- [ ] Final files are in `08_Delivery/`.
- [ ] Version number is set.
- [ ] Release notes are written.
- [ ] Known issues are documented.

## After Release

- [ ] Archive old versions if needed.
- [ ] Add next tasks to `TASKS.md`.
- [ ] Update roadmap.
